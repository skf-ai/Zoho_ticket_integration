# Courses, content, and access

## Course is not showing on the dashboard

1. Enrolment may still be pending approval. Ask the student to log out, wait a
   few minutes, and log back in.
2. Check they are looking at the right place — courses appear under **My
   courses** on the dashboard, and completed or hidden courses may be filtered
   out of the default view.
3. Some courses only open on a scheduled start date.

If the student was told they are enrolled and the course still does not appear
after a fresh login, raise a ticket — enrolment has to be fixed by the LMS admin.

## Videos or content will not load

1. Check the internet connection is stable; video needs more bandwidth than
   pages do.
2. Update the browser to the latest version.
3. Disable ad-blockers and privacy extensions for the site — these commonly block
   embedded video players.
4. Try a different browser or a private/incognito window.

If content fails to load on more than one device and network, raise a ticket —
the file itself may be broken or unpublished.

## Using a phone

Install the official Moodle app from the Play Store or App Store and enter
<LMS_URL> as the site address. Log in with the same username and password.

## Quizzes, assignments, and submissions

If a quiz will not open, check whether it has a scheduled availability window or
a time limit — these are set by the instructor. If a submission fails, check the
file size and format against what the assignment allows.

If a student loses work because a quiz or submission failed mid-attempt, raise a
ticket immediately and say clearly in the description that an attempt was lost —
this is time-sensitive, especially during exams.

## Grades and certificates

Grades and certificates appear inside the course once the instructor releases
them. Support cannot release them early. If a certificate is missing after the
course is complete and grades are published, raise a ticket.
